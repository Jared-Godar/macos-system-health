## Platform Support Matrix

macOS System Health targets the following platform combinations for v1.0:

| macOS Version | Apple Silicon | Intel | Test Status | Notes |
|---------------|---------------|-------|-------------|-------|
| 15 (Sequoia)  | ✓ | ✓ | ✓ Tested | Latest stable |
| 14 (Sonoma)   | ✓ | ✓ | ✓ Tested | Previous stable |
| 13 (Ventura)  | ✓ | ✓ | ✓ Tested | Older stable |
| 12 (Monterey) | ✓ | ✓ | ⚠️ Partial | EOL; minimal validation |

**Bash compatibility:** Bash 3.2+ (macOS default) and later.

**Architecture notes:**
- Apple Silicon validation is complete.
- Intel macOS validation is in progress ([#8](https://github.com/Jared-Godar/macos-system-health/issues/8)).
- Homebrew, Conda, pip, and `msmtp` are optional and auto-detected per run.

## Known Limitations

### Current (v1.0)

- **Email delivery:** Single recipient only via `msmtp`. Multi-recipient and provider plugins deferred to v1.1 ([#13](https://github.com/Jared-Godar/macos-system-health/issues/13)).
- **Log retention:** No automatic pruning; backups are unbounded. Configurable retention planned ([#7](https://github.com/Jared-Godar/macos-system-health/issues/7)).
- **Maintenance scope:** Does not upgrade Conda or pip packages. Upgradeable components are documented in the maintenance output.
- **Monitoring:** No JSON export in v1.0. Optional versioned JSON for monitoring tools deferred to v1.1 ([#10](https://github.com/Jared-Godar/macos-system-health/issues/10)).
- **Tool selection:** All tools (Homebrew, Conda, pip) run by default when present. Per-tool opt-in controls planned ([#11](https://github.com/Jared-Godar/macos-system-health/issues/11)).

### Known Issues

- [#24](https://github.com/Jared-Godar/macos-system-health/issues/24): Email multi-recipient edge case and unbounded backups (tracked for future release).

## Deferral Rationale

### v1.1 and Later

**[#10](https://github.com/Jared-Godar/macos-system-health/issues/10) — JSON output for monitoring tools**
Rationale: Monitoring integration is a post-release feature; v1.0 focuses on the core report-and-maintenance workflow. JSON export can be added without changing the CLI interface.

**[#13](https://github.com/Jared-Godar/macos-system-health/issues/13) — Notification providers**
Rationale: Multi-provider notification is complex and requires careful credential handling. v1.0 stabilizes the core tool and single-provider (`msmtp`) delivery; provider expansion deferred to v1.1.

**[#24](https://github.com/Jared-Godar/macos-system-health/issues/24) — Email multi-recipient and unbounded backups**
Rationale: Multi-recipient email and backup retention are enhancements; v1.0 establishes the single-recipient interface. Both deferred pending log retention design ([#7](https://github.com/Jared-Godar/macos-system-health/issues/7)).
