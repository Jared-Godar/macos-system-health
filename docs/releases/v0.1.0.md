# v0.1.0 release notes

The first public release provides a conservative macOS workstation health reporter with an explicit maintenance boundary.

## Highlights

- Read-only-by-default reporting for macOS, Homebrew, Conda, pip, disk usage, and environment inventories.
- Explicit maintenance mode limited to Homebrew update, upgrade, and cleanup plus Conda cache cleanup.
- Weekly report-only scheduling through a user LaunchAgent.
- Optional emailed reports through `msmtp`, with credentials kept outside the repository.
- Private-by-default logs and backups, per-user execution locking, and redaction of home-directory and Conda-base paths.
- Local pre-commit and GitHub Actions checks covering Bash syntax, ShellCheck, Actionlint, plist validation, smoke tests, whitespace, and secret scanning.

## Validation

The release was exercised on macOS in report, maintenance, and scheduled modes. Generated and emailed reports were checked for path disclosure, logs retained owner-only permissions, scheduler removal and reinstallation succeeded, and the complete automated quality suite passed.

## Known limitations

- Intel macOS remains untested.
- Package checks depend on the locally installed Homebrew, Conda, and pip versions.
- Warnings from package-manager diagnostic commands may require manual interpretation.
