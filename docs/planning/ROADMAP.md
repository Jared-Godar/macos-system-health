# Roadmap

## 1.0 release

- Define supported platforms, dependencies, exit codes, privacy requirements, and release acceptance criteria ([#6](https://github.com/Jared-Godar/macos-system-health/issues/6)).
- Publish an architecture decision record for report-versus-maintenance boundaries ([#9](https://github.com/Jared-Godar/macos-system-health/issues/9)).
- Add configurable log retention and maintenance dry-run controls ([#7](https://github.com/Jared-Godar/macos-system-health/issues/7)).
- Add per-tool opt-in and bounded command execution ([#11](https://github.com/Jared-Godar/macos-system-health/issues/11)).
- Validate the release candidate on Intel macOS; Apple Silicon validation is complete ([#8](https://github.com/Jared-Godar/macos-system-health/issues/8)).
- Define reproducible release artifacts, checksums, and signing or provenance guidance ([#12](https://github.com/Jared-Godar/macos-system-health/issues/12)).

## Proposed 1.1 release

- Add optional, versioned JSON output for monitoring tools ([#10](https://github.com/Jared-Godar/macos-system-health/issues/10)).
- Add notification providers behind a credential-free interface ([#13](https://github.com/Jared-Godar/macos-system-health/issues/13)).
